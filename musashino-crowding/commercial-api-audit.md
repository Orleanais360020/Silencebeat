# 商用・第三者API候補の監査

確認日: 2026-07-17

## 判定基準

このアプリの目的は、武蔵野線 東浦和 - 南流山間について、JR公式または許諾済みデータから空いている車両を出すこと。

そのため、以下を満たさないものは完成データ源にしない。

- JR公式、またはJR東日本から許諾された混雑データであること。
- 少なくとも列車単位のリアルタイム混雑であること。
- 号車別を名乗る場合は、普通車の号車別混雑が提供されていること。
- 公開Webアプリで表示する利用許諾があること。

## NAVITIME API

URL:

- https://api-sdk.navitime.co.jp/api/specs/api_guide/congestion.html
- https://api-sdk.navitime.co.jp/api/specs/examples/transit_congestion.html

確認内容:

- 乗換検索結果に混雑度を表示する `options=congestion` がある。
- 対応路線に武蔵野線が含まれる。
- 電車時刻表・混雑度情報は契約オプションが必要。
- サンプルは平均所要時間で、実時刻表利用は別途オプション。

評価:

- 契約可能な混雑API候補としては有力。
- JR公式速報値ではない。
- 号車別混雑ではない。
- 今回の「JRから取得した号車別速報値」の完成条件は満たさない。

## 駅すぱあと API

URL:

- https://ekispert.jp/products/api
- https://docs-mixway.ekispert.jp/list

確認内容:

- 経路探索、駅情報、鉄道運行情報などの法人向けAPIがある。
- リアルタイム経路探索系APIはある。
- 公開ページ上で鉄道混雑度API、号車別混雑APIは確認できない。

評価:

- 経路探索・時刻表・運行情報の候補。
- 混雑、特に号車別混雑の取得元としては未確認。

## ジョルダン / ジョルダンライブ!

URL:

- https://about.jorudan.co.jp/products/live/
- https://norikae.jorudan.co.jp/openapi/

確認内容:

- ジョルダンライブ! はユーザー投稿型の運行情報・混雑情報を表示する。
- 電車内の混雑度を5段階で表示する説明がある。
- 乗換案内オープンAPIは契約制。

評価:

- ユーザー投稿情報であり、JR公式速報値ではない。
- 号車別ではない。
- 今回の完成条件は満たさない。

## LINEヤフー / Yahoo!乗換案内

URL:

- https://www.lycorp.co.jp/ja/news/release/019792/
- https://blog-transit.yahoo.co.jp/tips/20251027_congestioninfo.html

確認内容:

- JR東日本と連携し、JR東日本のリアルタイムデータ連携基盤を活用したリアルタイム混雑情報を提供。
- 走行中列車の混雑度を5段階で表示。
- Yahoo!公式ブログでは「列車ごとの混雑度」のため車両によっては当てはまらない場合がある、と説明されている。

評価:

- JR東日本由来の正規データである点では最も近い。
- Yahoo!アプリ内の機能であり、外部Webアプリ向け公開APIは確認できない。
- 号車別ではなく列車単位。

## 結論

現時点で最も正規ルートに近いのは RT-DIP / ODPT 経由のJR東日本データ提供。

商用APIではNAVITIMEが混雑API候補だが、号車別JR速報値ではない。駅すぱあと・ジョルダンは、公開情報だけでは今回の仕様を満たす混雑APIを確認できない。
