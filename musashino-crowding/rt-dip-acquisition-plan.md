# RT-DIP / ODPT 経由の取得計画

確認日: 2026-07-17

## 目的

武蔵野線 東浦和 - 南流山間の混雑情報を、JR東日本公式または許諾済みのルートで取得し、静的Webアプリに投入する。

## 現時点で確認できた正規ルート

JR東日本の「リアルタイムデータ連携基盤（RT-DIP）」が、列車の遅れ・在線位置などのリアルタイムデータを集約し、交通案内サービス提供事業者へ有償提供する基盤として公開説明されている。

根拠:

- JR東日本プレスリリース「JR東日本は『リアルタイムデータ連携基盤』を構築し、各事業者とのデータ連携を推進します」
- LINEヤフープレスリリース「Yahoo!乗換案内、JR東日本と連携し、いま走行している列車の混雑状況がわかる『リアルタイム混雑情報』の提供を開始」
- 公共交通オープンデータ協議会の説明では、RT-DIP は有償データ流通スキームの一部として機能予定とされている。

## 重要な制約

### 1. 公開Web APIではない

RT-DIP は、誰でもキーなしで取得できる公開APIではない。交通案内サービス提供事業者などへの有償データ提供が前提。

### 2. Yahoo!乗換案内で確認できるのは列車単位

LINEヤフーの説明では、JR東日本の各路線において、乗車予定の列車がどの程度混雑しているかを5段階で表示する機能と説明されている。

普通車の号車別速報値までは確認できていない。

### 3. JR東日本アプリの号車別は限定的

JR東日本コンサルタンツの説明では、山手線はリアルタイムで列車位置情報、車両別の車内温度と混雑度を確認できるとされている。

武蔵野線について、普通車の号車別混雑が提供されるとは確認できていない。

## 取得するために必要な確認

RT-DIP / ODPT / JR東日本側に、次を確認する必要がある。

1. 武蔵野線が RT-DIP のリアルタイム混雑情報対象か。
2. 東浦和 - 南流山間の列車単位混雑が取得できるか。
3. 号車別混雑データが存在するか。
4. 存在する場合、外部Webアプリで表示できる契約が可能か。
5. 静的サイトから直接呼べるか、APIキー保護のため中継サーバーが必要か。
6. 二次利用表示、免責、更新間隔、キャッシュ、再配信禁止などの条件。

## アプリ側の受け入れ形式

現在の公開アプリは、次のJSONを受け取れるようにしてある。

```json
{
  "source": "JR East RT-DIP licensed feed",
  "licensed": true,
  "updatedAt": "2026-07-17T08:00:00+09:00",
  "trains": [
    {
      "trainId": "example",
      "line": "武蔵野線",
      "serviceName": "普通",
      "direction": "higashiurawa-minaminagareyama",
      "fromStation": "東浦和",
      "toStation": "南流山",
      "departureTime": "08:12",
      "crowdingLevel": 3,
      "cars": [
        { "carNumber": 1, "crowdingLevel": 2 },
        { "carNumber": 2, "crowdingLevel": 3 }
      ]
    }
  ]
}
```

`cars` がない場合は、アプリは「号車別判定不可」と表示する。推定で補完しない。

## 問い合わせ文面

件名:

武蔵野線リアルタイム混雑情報の外部Webアプリ利用可否について

本文:

```
公共交通オープンデータ協議会 / RT-DIP ご担当者様

武蔵野線 東浦和駅 - 南流山駅間の利用者向けに、列車混雑情報を表示するWebアプリを検討しています。

JR東日本のリアルタイムデータ連携基盤（RT-DIP）または公共交通オープンデータセンター経由で、以下のデータを取得・表示できる契約または利用枠があるか確認させてください。

1. 武蔵野線の走行中列車のリアルタイム混雑情報
2. 東浦和 -> 南流山 / 南流山 -> 東浦和の方向別列車情報
3. 列車単位の5段階混雑情報
4. 可能であれば普通車の号車別混雑情報
5. Webアプリで一般利用者へ表示する場合の利用条件、費用、更新間隔、表示義務、再配信制限

現時点の実装では、許諾済みフィードであることを示す `licensed: true` がある場合のみ表示し、推定値は表示しない設計にしています。

上記データの提供可否、申込方法、仕様書の開示条件をご教示いただけますと幸いです。
```
